# 架构设计文档

> 记录时间：2026-06-20

## 项目架构

### 当前架构

```
用户请求
    │
    ▼
manage.py / WSGI / ASGI
    │
    ▼
config/urls.py  (URL 路由分发)
    │
    ▼
Django Views  (待创建)
    │
    ▼
Django Models  (待创建)
    │
    ▼
SQLite 数据库  (开发环境)
```

### 目录结构说明

```
rance-blog/
│
├── config/                 ← Django 项目配置模块
│   ├── settings.py         ─ 全局配置（数据库、中间件、App 注册）
│   ├── urls.py             ─ 根 URL 路由配置
│   ├── asgi.py             ─ ASGI 入口（异步部署用）
│   └── wsgi.py             ─ WSGI 入口（传统部署用）
│
├── apps/                   ← 后续存放 Django apps（待创建）
│   ├── blog/               ─ 博客核心功能（文章、分类、标签）
│   └── ...                 ─ 其他 App
│
├── templates/              ← 项目级模板目录（待创建）
├── static/                 ← 静态文件（CSS/JS/图片）（待创建）
├── media/                  ← 用户上传文件（待创建）
│
├── doc/                    ─ 项目文档
├── manage.py               ─ Django 命令行入口
├── pyproject.toml          ─ Python 项目配置
└── uv.lock                 ─ 依赖锁定
```

## 核心模块（规划）

### 1. 博客文章模块 (`apps/blog`)
- 文章 Model: 标题、正文、摘要、封面图、状态、发布时间等
- 分类 Model: 名称、slug、描述
- 标签 Model: 名称、slug（多对多关系）
- 文章管理器: 分页、按分类/标签筛选、全文搜索

### 2. 用户模块（Django 内置 auth + 扩展）
- 使用 Django 自带 User 模型
- 可选扩展 Profile 模型（头像、个人简介等）

### 3. 评论模块（规划中）
- 支持文章评论
- 嵌套回复
- 可选的审核机制

## 设计原则

- **App 化**: 功能模块以 Django app 的方式组织，保持低耦合
- **配置分离**: 开发/生产配置分离（通过环境变量或配置文件）
- **DRY**: 复用 Django 内置功能（Admin、Auth、Session 等）
