# 部署文档

> 记录时间：2026-06-20

## 环境要求

| 组件 | 要求 |
|------|------|
| 操作系统 | Rocky Linux 9 / 10（全新安装） |
| Python | >= 3.14（通过 pyenv 安装） |
| 包管理 | uv（Python）/ dnf（系统） |
| Web 服务器 | Nginx（反向代理） |
| 应用服务器 | Gunicorn（WSGI） |
| 进程管理 | Supervisor |
| 数据库 | SQLite（默认，小规模使用） |

## 快速部署（Docker Compose）

### 前提

- 安装 Docker 和 Docker Compose

### 步骤

```bash
# 1. 克隆项目
git clone https://github.com/gallantsa/rance-blog.git
cd rance-blog

# 2. 创建环境变量文件
mkdir -p .envs
cat > .envs/.production << EOF
DJANGO_SECRET_KEY=your-secure-random-key
EOF

# 3. 构建并启动
docker compose -f production.yml build
docker compose -f production.yml up -d

# 4. 创建管理员
docker exec -it rance-blog uv run python manage.py createsuperuser
```

### 停止

```bash
docker compose -f production.yml down
```

### 架构

```
Nginx (80/443) ──► Gunicorn (8000) ──► SQLite
     │                    │
     ▼                    ▼
  /app/static         /app/database
  (volume)            (volume)
```

- Nginx 通过共享 volume `static` 提供静态文件服务
- Django 通过共享 volume `database` 持久化 SQLite 数据库

## 传统部署（Rocky Linux）

```bash
# 1. 上传部署脚本到服务器
scp deploy_rocky.sh root@your-server:/root/

# 2. SSH 登录服务器
ssh root@your-server

# 3. 设置环境变量
export REPO_URL="https://github.com/gallantsa/rance-blog.git"
export DJANGO_SECRET_KEY="$(openssl rand -hex 40)"

# 4. 执行部署
chmod +x deploy_rocky.sh
sudo ./deploy_rocky.sh
```

> ⚠️ 部署全程约 15-30 分钟（取决于服务器性能，主要耗时在编译 Python 3.14）

## 手动部署步骤

如不适用一键脚本，也可按以下步骤手动部署。

### 1. 系统依赖

```bash
# 更新系统
dnf update -y
dnf install -y epel-release

# 安装编译工具
dnf groupinstall -y "Development Tools"
dnf install -y \
    gcc gcc-c++ make wget git \
    openssl-devel bzip2-devel libffi-devel zlib-devel \
    readline-devel sqlite-devel ncurses-devel xz-devel tk-devel \
    libxml2-devel libxslt-devel bzip2 \
    nginx supervisor
```

### 2. 安装 Python 3.14

```bash
# 安装 pyenv
curl -fsSL https://pyenv.run | bash

# 配置环境变量
export PYENV_ROOT="$HOME/.pyenv"
export PATH="$PYENV_ROOT/bin:$PATH"
eval "$(pyenv init -)"

# 安装 Python 3.14.6（编译耗时较长）
pyenv install 3.14.6
pyenv global 3.14.6
python --version  # 确认: Python 3.14.6
```

### 3. 安装 uv

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
export PATH="$HOME/.local/bin:$PATH"
uv --version
```

### 4. 克隆项目并安装依赖

```bash
git clone -b master https://github.com/gallantsa/rance-blog.git /var/www/rance-blog
cd /var/www/rance-blog
uv sync --frozen
```

### 5. 配置环境变量

```bash
export DJANGO_SECRET_KEY="your-secure-random-key-here"
export DJANGO_SETTINGS_MODULE="config.settings.production"
```

### 6. 数据库迁移与静态文件

```bash
uv run python manage.py migrate --noinput
uv run python manage.py collectstatic --noinput
uv run python manage.py createsuperuser
```

### 7. 配置 Supervisor

创建 `/etc/supervisord.d/rance-blog.ini`：

```ini
[program:rance-blog]
command=/var/www/rance-blog/.venv/bin/gunicorn config.wsgi:application \
    --bind 127.0.0.1:8000 \
    --workers 4 \
    --timeout 120 \
    --access-logfile /var/log/gunicorn/access.log \
    --error-logfile /var/log/gunicorn/error.log
directory=/var/www/rance-blog
user=root
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
environment=DJANGO_SECRET_KEY="your-key",DJANGO_SETTINGS_MODULE="config.settings.production"
stdout_logfile=/var/log/gunicorn/supervisor.log
stderr_logfile=/var/log/gunicorn/supervisor_err.log
```

启动：

```bash
supervisorctl reread
supervisorctl update
supervisorctl start rance-blog
```

### 8. 配置 Nginx

创建 `/etc/nginx/conf.d/rance-blog.conf`：

```nginx
server {
    listen      80;
    server_name _;
    charset     utf-8;

    client_max_body_size 10M;

    location /static {
        alias /var/www/rance-blog/static;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    location /media {
        alias /var/www/rance-blog/media;
    }

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```bash
# 移除默认站点
rm -f /etc/nginx/conf.d/default.conf

# 语法检查
nginx -t

# 启动
systemctl enable nginx
systemctl restart nginx
```

### 9. 防火墙

```bash
firewall-cmd --permanent --add-service=http
firewall-cmd --permanent --add-service=https
firewall-cmd --reload
```

### 10. 配置 HTTPS（Let's Encrypt + Certbot）

免费获取 TLS 证书，开启 HTTPS。

#### 安装 Certbot

```bash
dnf install -y certbot python3-certbot-nginx
```

#### 获取并配置证书

```bash
# 将 nginx 配置中的 server_name 改为你的域名
sed -i 's/server_name _;/server_name your-domain.com;/' /etc/nginx/conf.d/rance-blog.conf
nginx -t && systemctl reload nginx

# 自动获取证书并修改 nginx 配置
certbot --nginx -d your-domain.com --non-interactive --agree-tos -m admin@your-domain.com

# 证书自动续期（Let's Encrypt 证书 90 天有效期）
certbot renew --dry-run
```

Certbot 会自动修改 Nginx 配置，添加 SSL 相关指令并将 HTTP 重定向到 HTTPS。

> 注意：`server_name _;` 无法通过域名验证，需先改为实际域名。

#### 修复 Mixed Content 警告

启用 HTTPS 后，模板中所有引用的外部资源必须使用 `https://` 协议，否则浏览器会拦截。

已处理的 CDN 资源：

```html
<!-- ❌ 混合内容（HTTP 引用 → 浏览器拦截） -->
<link rel="stylesheet" href="http://code.ionicframework.com/ionicons/...">

<!-- ✅ 已修复为 HTTPS -->
<link rel="stylesheet" href="https://code.ionicframework.com/ionicons/...">
```

涉及文件：`templates/base.html`

## 部署脚本参考

一键部署脚本 `deploy_rocky.sh` 位于项目根目录，自动执行以上全部步骤。

脚本内部流程：

```
check_prerequisites       → 权限检查 + DJANGO_SECRET_KEY 校验
install_system_packages   → dnf 安装编译工具 / nginx / supervisor
install_python            → pyenv 编译 Python 3.14.6
install_uv                → 安装 uv 包管理器
clone_project             → git clone 到 /var/www/rance-blog
install_dependencies      → uv sync 安装依赖
configure_environment     → 写入 DJANGO_SECRET_KEY
setup_django              → migrate + collectstatic + createsuperuser
configure_supervisor      → 写入 Supervisor 配置
configure_nginx           → 写入 Nginx 反向代理配置
configure_firewall        → 开放 HTTP/HTTPS
start_services            → 启动 supervisor + nginx
```

## 配置分离说明

项目使用 split settings 模式：

```
config/settings/
├── common.py          # 公共配置（数据库、App、中间件）
├── local.py           # 开发配置（DEBUG=True）
└── production.py      # 生产配置（DEBUG=False，密钥从环境变量读取）
```

- 开发：`manage.py` 默认加载 `config.settings.local`
- 生产：`wsgi.py` 默认加载 `config.settings.production`

## 生产环境注意事项

### 安全性

1. **Django Secret Key**：使用强随机密钥，`openssl rand -hex 40` 生成
2. **DEBUG = False**：生产环境务必关闭调试模式
3. **ALLOWED_HOSTS**：限制为实际域名，如 `['blog.example.com']`
4. **HTTPS**：部署后参考 §10 使用 Let's Encrypt + Certbot 免费配置 HTTPS。注意将 `server_name` 改为实际域名，并确保所有外部 CDN 资源使用 `https://` 协议避免 Mixed Content 警告。

### 性能

1. **Gunicorn workers**：`2 * CPU核心数 + 1`，例如 2 核 CPU 配置 5 个 workers
2. **Nginx 静态文件缓存**：已配置 `expires 30d`
3. **数据库**：SQLite 适合小流量博客，高并发建议切换 PostgreSQL
