# 项目中遇到的问题解决

## Git 相关

### 从 Git 暂存区移除文件但不删除本地

项目初始化阶段如果无意添加了不必要的文件到 git 暂存区，可以使用以下命令将这些文件从暂存区移除，而不删除工作目录中的文件：

```bash
# 将.idea目录从git暂存区移除，但保留在本地
git rm --cached -r .idea/
```

## Django / Python 开发问题

### 1. `markdown.markdown()` 误用导致 `AttributeError`

**现象**：访问文章详情页时抛出 `AttributeError: 'str' object has no attribute 'convert'`

**根因**：`markdown.markdown()` 是快捷函数，返回 HTML 字符串，但代码将其当作 Markdown 实例调用了 `.convert()` 和 `.toc`。

```python
# ❌ 错误写法
md = markdown.markdown(extensions=[...])  # 返回 str（HTML）
post.body = md.convert(post.body)         # AttributeError：str 没有 .convert()
post.toc = md.toc                          # AttributeError：str 没有 .toc
```

**解决**：改用 `markdown.Markdown()` 类构造器，获得对象实例：

```python
# ✅ 正确写法
md = markdown.Markdown(extensions=[...])   # 返回 Markdown 实例
post.body = md.convert(post.body)          # ✅ 有 .convert() 方法
post.toc = md.toc                           # ✅ 有 .toc 属性（TocExtension注入）
```

| API | 返回类型 | 用途 |
|-----|---------|------|
| `markdown.markdown(text, ...)` | `str` | 快捷函数，一次性 Markdown → HTML |
| `markdown.Markdown(extensions=...)` | `Markdown` 实例 | 类构造器，需要获取 `.toc` 等附加属性时使用 |

### 2. `{% load %}` 缺失导致 `TemplateSyntaxError`

**现象**：访问博客页面时抛出 `TemplateSyntaxError: Invalid block tag on line: 'static'`

**根因**：重构侧边栏为自定义模板标签时，`base.html` 的 `{% load static %}` 被误替换为 `{% load blog_extras %}`，但模板头部仍在使用 `{% static '...' %}` 引用静态文件。

```html
<!-- ❌ 错误 — 只加载了自定义标签库 -->
{% load blog_extras %}

<!-- 以下 {% static %} 找不到标签库 → TemplateSyntaxError -->
<link rel="stylesheet" href="{% static 'blog/css/bootstrap.min.css' %}">
```

**解决**：同时加载两个标签库（先后顺序均可）：

```html
<!-- ✅ 正确 — 同时加载两个标签库 -->
{% load static %}
{% load blog_extras %}
```

**注意**：Django 模板中每个用到的标签库都需要 `{% load %}` 声明，加载多个可在多行中分别写，也可合并为 `{% load static blog_extras %}`。
