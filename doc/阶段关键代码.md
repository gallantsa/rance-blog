## 2. manage.py（Django 入口）

由 `django-admin startproject config .` 自动生成，位于项目根目录。

```python
#!/usr/bin/env python
"""Django's command-line utility for administrative tasks."""
import os
import sys


def main():
    """Run administrative tasks."""
    os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'config.settings')
    try:
        from django.core.management import execute_from_command_line
    except ImportError as exc:
        raise ImportError(
            "Couldn't import Django. Are you sure it's installed and "
            "available on your PYTHONPATH environment variable? Did you "
            "forget to activate a virtual environment?"
        ) from exc
    execute_from_command_line(sys.argv)


if __name__ == '__main__':
    main()
```

关键点：
- 设置 `DJANGO_SETTINGS_MODULE=config.settings`，指向 `config/settings.py`
- 通过 `uv run python manage.py <command>` 执行

## 7. blog/models.py（数据模型）

定义了 3 个模型：Category（分类）、Tag（标签）、Post（文章）。

### Category（分类）

```python
class Category(models.Model):
    name = models.CharField(max_length=100)
```

- 只有 `name` 一个字段，用作文章分类
- 与 Post 为一对多关系（ForeignKey）

### Tag（标签）

```python
class Tag(models.Model):
    name = models.CharField(max_length=100)
```

- 只有 `name` 一个字段，用作文章标签
- 与 Post 为多对多关系（ManyToManyField）

### Post（文章）

```python
class Post(models.Model):
    title = models.CharField(max_length=70)
    body = models.TextField()
    created_time = models.DateTimeField()
    modified_time = models.DateTimeField()
    excerpt = models.CharField(max_length=200, blank=True)
    category = models.ForeignKey(Category, on_delete=models.CASCADE)
    tags = models.ManyToManyField(Tag, blank=True)
    author = models.ForeignKey(User, on_delete=models.CASCADE)
```

| 字段 | 类型 | 说明 |
|------|------|------|
| title | CharField(70) | 文章标题 |
| body | TextField | 文章正文，大段文本使用 TextField |
| created_time | DateTimeField | 创建时间 |
| modified_time | DateTimeField | 最后修改时间 |
| excerpt | CharField(200, blank=True) | 文章摘要，允许为空 |
| category | ForeignKey → Category | 所属分类，级联删除 |
| tags | ManyToManyField → Tag | 关联标签，允许为空 |
| author | ForeignKey → User | 作者，关联 django auth.User，级联删除 |

## 9. 模型添加 `__str__` 方法（blog/models.py）

```python
# commit daabac5
class Category(models.Model):
    name = models.CharField(max_length=100)

    def __str__(self):
        return self.name


class Tag(models.Model):
    name = models.CharField(max_length=100)

    def __str__(self):
        return self.name


class Post(models.Model):
    # ... 字段定义 ...

    def __str__(self):
        return self.title
```

- `__str__` 是 Python 对象字符串表示方法
- Django Admin 后台和 shell 中通过此方法显示模型实例
- 各模型分别返回其有意义的字段：Category → name、Tag → name、Post → title

## 10. 生成数据库迁移文件

```bash
# commit daabac5
uv run python manage.py makemigrations
uv run python manage.py migrate
```

生成的迁移文件 `blog/migrations/0001_initial.py` 包含了三个模型的创建操作：
- `CreateModel(name='Category', ...)`
- `CreateModel(name='Tag', ...)`
- `CreateModel(name='Post', ...)` — 含 author（FK → User）、category（FK → Category）、tags（M2M → Tag）

执行 `migrate` 后在 SQLite 中创建了 4 张新表：
- `blog_category` — 分类表
- `blog_tag` — 标签表
- `blog_post` — 文章表
- `blog_post_tags` — 文章-标签多对多关联表
